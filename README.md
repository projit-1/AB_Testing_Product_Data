# A/B Testing Analysis for E-Commerce Website

## 📌 Project Overview

This project demonstrates a comprehensive **A/B Testing Analysis** conducted on a simulated e-commerce website to determine whether a new website variant (Variant B) outperforms the existing design (Variant A). The analysis evaluates user behavior, conversion performance, revenue generation, engagement metrics, and statistical significance using Python and common data analysis libraries. The notebook walks through the complete A/B testing workflow, from loading and exploring the dataset to calculating business metrics and validating results with hypothesis testing. The dataset contains user-level information such as device type, country, traffic source, session duration, pages viewed, conversion status, order value, and revenue, enabling detailed performance comparisons across multiple user segments. 

---

## 🎯 Objectives

* Compare the performance of Variant A and Variant B.
* Measure conversion rate and revenue differences.
* Analyze user engagement metrics.
* Evaluate performance across devices, countries, and traffic sources.
* Validate findings using statistical hypothesis testing.
* Recommend the better-performing website version based on data.

## 📈 Key Insights

The analysis indicates that **Variant B consistently outperforms Variant A** across several business metrics. Variant B achieves a higher conversion rate, generates greater total revenue, and produces a positive conversion lift. Segment analysis further shows improved performance across different devices, countries, and traffic sources. Statistical testing confirms that the improvement in conversion rates is significant, providing evidence that the observed differences are unlikely to be due to random chance, while some engagement metrics such as session duration show no statistically significant difference.
